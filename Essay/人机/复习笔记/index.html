    关注思维过程与推理 像人一样思考 合理地思考
    强调行为 像人一样行动 合理地行动

    智能体 agent

    定义

    能够感知和动作的实体，即一个从感知序列到动作的函数 $$ f:P^* \to A $$ 计算能力有限使得最完美的智能体不可实现，只能针对给定机器资源设计最好的程序

    任何独立的能够思想并可以同环境交互的实体都可以抽象为智能体

    智能体与环境

    通过传感器感知环境，通过执行器影响环境

    感知信息：表示任何时刻Agent的感知输入。 感知序列：Agent收到的所有输入数据的完整历史。 Agent函数：将任意给定感知序列映射到Agent的动作。可以描述Agent的行为。

    Agent在任何时刻的行动选择，取决于到该时刻为止的整个感知序列

    智能体函数&智能体程序

    Agent函数：从外部反映了Agent的特性，是一个抽象的数学表示

    Agent程序：从Agent内部来看，Agent函数是通过Agent程序实现的，是Agent函数在某种物理体系上的具体实现

    Agent函数可以通过列表的形式表现：感知序列\(\to\)行动

    理性智能体

    定义：对于每一个可能的感知序列，根据已知的感知序列和内建的先验知识，理性Agent选择能使性能指标的期望值最大化的动作。

    性能度量：通常由理性Agent的设计者给出，根据实际在所处的环境中希望得到的结果来设计度量，而不是根据智能体应该表现的行为。

    • 当把Agent置于一个环境中后，它将针对收到的感知信息产生动作序列。
    • 该动作序列引起环境历经一个状态序列。
    • 如果该状态序列是想要的，则Agent的性能良好。

    理性的判断：任务环境PEAS

    （1）定义成功标准的性能度量 performance （2）Agent对环境的先验知识 environment （3）Agent可以执行的动作 actuators （4）Agent的感知序列 sensors

    真实的世界是部分可观，随机的，延续式的，动态的，连续的，多智能体的

    智能体的分类

    四种基本智能体的结构类型：

    • 简单反射型

    基于当前感知选择行动，忽略感知历史

    • 基于模型的简单反射型

    使用内部模型记录世界的当前状态，按反射型智能体方式选择行动

    • 基于目标型

    智能体追踪记录世界的状态、要达到的目标，并选择导致达成目标的行动。既追踪记录世界的状态，又记录要达到的目标集

    • 基于效用型

    更普适的度量：效用函数

    问题类型

    • 单一状态问题：确定的动作后果，完全可观 智能体确切知道自己将处于什么状态；解是一个序列

    • 构造问题：确定的动作后果、部分可观

    智能体不知道自己处于什么状态；解（如果存在的话）是一个序列

    • 偶然事件类型：不确定的动作后果、部分可观

    但是，感知提供关于目前状态的最新信息；解是树或是一种策略。

    • 探索性问题：对状态空间一无所知

    搜索

    搜索的主要过程（三要素）

    • 状态空间（state space）：

    从初始或目的状态出发，并将它作为当前状态。

    • 后继函数（successor function with actions and costs）：

    扫描操作算子集，将适用当前状态的一些操作算子作用于当前状态而得到新的状态，并建立指向其父结点的指针。

    • 初始状态和目标测试（start state and goal test）：

    解是一个行动序列，将初始状态转换成目标状态

    搜索算法的性能度量

    通过以下四种途径来判断算法的性能：

    • 完备性：有解时能否保证找到解 • 最优性：这个搜索策略是否能找到最优解 • 时间复杂度：根据搜索过程中产生的节点数目来度量，即找到解所花的时间 • 空间复杂度：在执行搜索的过程中需要的内存多少

    问题的难度（复杂度）常常转化为状态空间图的大小：

    • b：branching factor 分支因子，节点后继的最大个数 • d：solution depth 最浅目标节点的深度（最优解所在的深度） • m：maximum depth 状态空间中所有路径的最大长度（整个图的最大深度） 可能是∞

    时间复杂性：搜索所产生的节点个数 空间复杂性：存储的最大节点个数

    算法的有效性（效率）：

    可以只考虑搜索耗散(代价函数,cost),通常取决于时间复杂度

    也可以考虑总耗散，即时间复杂度+空间复杂度

    一致代价搜索

    对任何单步代价函数都是最优的算法

    总是搜索总成本最小的节点

    e.g.：

    一致代价搜索的性能；

    • 完备性：一致代价搜索对解路径的步数并不关心，只关心路径总代价。所以，如果存在零代价行动就可能陷入死循环。如果每一步的代价都大于等于某个小的正常数，那么一致代价搜索是完备的。从算法中也可以看到，一致搜索将地图中相关联的路径都入了队列，不存在遗漏的点线，所以如果目标结点在图中，最终总会找到一条路径到达目标结点。
    • 最优性：一致代价搜索是最优的。一致代价搜索目标检测应用于结点被选择扩展时，而不是在结点生成的时候，因为第一个生成的目标结点可能在次优路径上。而且如果边缘中的结点有更好的路径到达该结点那么会引入一个测试。简单来说，优先队列保证了每一个出队扩展的结点都是最优的，如此得到了一个最优的路径。
    • 时间和空间复杂度∶一致代价搜索由路径代价而不是深度来导引。引入C 表示最优解的代价，假设每个行动的代价至少为e，那么最坏情况下，算法的时间和空间复杂度为\(O(b\cdot(1+\frac{C}{e}))\)

    有可能陷入无限循环（遇到具有能返回同一状态的零耗散行动的节点时）;如果规定每一步的耗散都大等于某个小的正值常数，那么就能保证完备性和最优性。

    代价一致搜索由路径的耗散引导不是由深度引导，其复杂度与b和最优解的耗散值有关。

    IDS就是在DLS的基础上，不断的将常数l扩大，从而增加搜索深度。

    迭代加深的深度优先搜索：其经常和深度优先搜索结合使用以确定最好的深度界限，其基本原理是：不断增加深度限制，从0开始，接下来为1、2、3,...，依此类推，直到找到目标。显然，当深度界限达到d时，即最浅的目标结点深度，就能找到目标结点。

    BFS的衍生：双相搜索 BS

    BS的假设是我们不仅可以从初始节点出发，同时还可以从目标节点出发，从而实现双向的搜索。BS的内部往往是由BFS实现的，如果从初始节点和从目标节点的搜索都是BFS，可以保证算法的最优。

    实际上，BS就是将BFS画的半径为d的一个大圆，换成半径为d/2的两个小圆。

    双向搜索的实现方式是：将目标测试替换为检查两个方向的搜索边缘结点集是否相交。

    最好优先搜索BFS介绍

    基于广搜，与一致代价搜索类似，但是核心思想不同

    最佳优先搜索：使用一个评价函数\(f(n)\)给每个节点估计他们的希望值。每次搜索时优先扩展最有希望的未扩展节点（评估值最小的节点）

    一致代价搜索：使用一个路径耗散函数$g(n) $扩展路径消耗最低的节点,总是搜索总成本最小的节点

    启发函数h(n)= 从节点n代表的状态到目标状态的路径耗散的最小估计值

    贪婪最好优先搜索:\(f(n)=h(n)\)

    每次搜索会拓展使得\(h(n)\)最小的节点，类似于DFS

    算法性能：

    该算法的问题：

    算法性能

    \(A^*\)搜索 \(f(n)=g(n)+h(n)\)

    g(n)从初始节点到节点n的实际路径耗散

    h(n)从节点n到目标节点的最小路径耗散估计

    f(n)从初始节点经过节点n到达目标节点的最小路径耗散估计

    如果h从不高估（\(\le\)）到达目标的最低路径耗散值，则称h是可容纳启发式

    如果h是可容许的，那么\(A^*\)算法是最优的

    算法性能：

    但\(A^*\)对存储空间要求很高

    \(A^*\)算法的拓展:迭代深入\(A^*\)算法(\(IDA^*\))

    截断值是超过上一次迭代截断值的节点中最小的f耗散值

    • 不需要判重，不需要排序
    • 空间需求减少

    递归最好优先搜索(RBFS) 这个地方没看懂

    尝试模拟标准的最佳优先搜索而只需线性空间。 1. 记录当前节点的祖先可得到的最佳可替换路径的f值。 2. 如果当前的f值超过了这个限制，则递归将转回到替换路径。 3. 向上回溯改变f值到它的孩子的最佳f 值 4. 重复扩展这个上个节点，因为仍有可能存在较优解。

    如果h是可采纳的，则有最优性

    空间复杂度：\(O(bd)\)

    时间复杂度：取决于h的精度和最佳路径变换的次数

    不完全信息博弈

    在不完全信息博弈里，参与人并不完全清楚有关博弈的一些信息。大多数纸牌游戏是不完全信息博弈。在桥牌里，你并不知道你对面伙伴手中的牌，也并不知道坐在左右两位对手手里的牌。你在作决策时，必须对其他三位手中的牌做一个估计，而没有确切的信息。

    思想： 对每种牌局计算每个行为的最小最大值，然后选择对所有牌局期望值最高的行为。如果某个行为对所有的牌局都是最优的，则它是最优的。

    命题逻辑

    原子命题：单个命题词组成，每个命题词代表一个真/假的命题

    文字：原子命题及其否定

    互补文字：一个文字是另一个文字的否定

    子句：文字的析取式\(\or\)

    子句集：n个子句的合取\(\and\)

    归并：去除文字的多余副本

    语法：（按照优先级列出）

    $\neg非 \qquad \and合取 \qquad \or析取 \qquad \Rightarrow蕴含 \qquad \Leftrightarrow当且仅当 $

    命题逻辑的主要研究对象是公式

    合式公式 (well-formed formula) 简称公式，记作 wff

    • 一个命题变量 p 是一个 wff
    • 若 A 是 wff ，则 (¬A) 也是 wff
    • 若 A, B 是 wff ，则 (A\(\and\)B), (A$\or $B), (A→B), (A↔B) 也是 wff
    • 当且仅当有限次使用上述规则得到的才是 wff 。

    区分⇒（蕴含）和 ╞ （蕴涵)

    命题逻辑的推理

    有效性 validity 定义： 一个语句是有效的，如果在所有的模型中它都为真。

    可满足性 satisfiability 定义：一个语句是可满足的，如果对于部分模型它是真值。

    α 是有效的 当且仅当¬α不可满足

    α 是可满足的当且仅当¬α不是有效的

    推导和证明

    假言推理规则：

    $\frac{\alpha \rightarrow \beta ，\alpha }{\beta } $

    消去合取词：

    $\frac{\alpha \wedge \beta }{\alpha } $

    逻辑系统的单调性：只要在知识库里面发现了合适的前提，就可以应用推理规则，但与知识库的其余内容无关

    命题逻辑的每个语句逻辑上等价于文字析取式的合取式CNF

    如何证明\(KB \models \alpha ?\)

    受限子句 限定子句 确定子句:指恰好只含有一个正文字的析取式

    霍恩子句:至多只有一个正文字的文字析取式

    没有正文字的析取式也是 Horn 子句，被称为 目标子句

    谓词逻辑

    命题逻辑的局限：

    • 无法把不同事物间的共同特征表达出来。
    • 无法把所描述的客观事物的结构及逻辑特征反映出来。

    命题内部逻辑结构的分解：

    把判断中的具体内容抽出，称为个体；剩下的判断即为谓词

    一阶逻辑的结构：论域、个体、个体间的关系，作用于个体的函数

    在一阶逻辑中，被研究对象构成的非空集称为论域 ；论域中的每个元素称为 个体（对象），通常用小写字母表示

    谓词：一阶形式语言中用于指称论域中 个体的性质 或者 个体之间关系 的形式符号，大写字母

    • 一元谓词：个体的性质；

    • 二元谓词 多元谓词 ：个体的关系

    • 个体的位置：空位；具体化：构成意义完整的语句
    • 空位的数目-谓词的元数-n元谓词

    函数： 为了表达个体之间的对应关系，引入数学中函数概念和记法。小写字母

    x 的父亲 father（张三）

    在 n 元谓词 P(x1 X2 ，，…xn) 中，若每个个体均为 常量、变元 或 函数 则称它为一阶谓词。如果某个个体本身又是一个一阶谓词，则称它为二阶谓词，如此类推。

    量词：全称量词、存在量词

    约束变量、量词辖域、自由变量
