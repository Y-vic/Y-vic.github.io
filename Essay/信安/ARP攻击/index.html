    ARP攻击

    ARP攻击原理

    ARP（Address Resolution Protocol）攻击是一种网络攻击，利用了ARP协议的工作机制漏洞，进行ARP攻击，首先需要了解ARP协议原理。ARP是一种用于将IP地址解析为MAC地址的协议，它在局域网（LAN）上工作，通过ARP协议，网络中的设备能够动态地维护IP地址和MAC地址之间的映射关系。

    ARP协议原理：

    1. IP地址与MAC地址的映射：每个设备在网络上都有唯一的MAC地址和IP地址。MAC地址是设备的物理地址，用于在局域网上唯一标识设备。IP地址是设备在网络上的逻辑地址，用于在全球范围内唯一标识设备。
    2. ARP请求：当一个主机需要将IP地址解析为目标主机的MAC地址时，它会发送一个ARP请求广播消息到局域网上的所有设备。ARP请求包含了源主机的IP地址和MAC地址，并询问谁拥有目标IP地址所对应的MAC地址。
    3. ARP响应：拥有目标IP地址的主机收到ARP请求后，会发送一个ARP响应消息，其中包含它的IP地址和MAC地址。这样，源主机就可以通过接收到的ARP响应来获取目标主机的MAC地址。
    4. ARP缓存：源主机收到ARP响应后，将目标IP地址和对应的MAC地址存储在本地的ARP缓存中。这样，在将来与目标主机通信时，源主机可以直接从ARP缓存中获取目标主机的MAC地址，而无需再次发送ARP请求。
    5. ARP缓存更新：ARP缓存中的条目具有一定的生命周期。如果在一段时间内没有与目标主机通信，缓存中的条目可能会过期。当源主机需要与过期的目标主机通信时，它会发送一个新的ARP请求来获取最新的MAC地址。

    ARP协议使得设备能够准确地将网络流量发送到目标设备，并实现有效的通信。但同时，ARP协议也容易受到ARP攻击的威胁。

    因为ARP协议是建立在信任局域网内所有节点的基础上的，他的效率很高。但是不安全。它是无状态的协议。他不会检查自己是否发过请求包，也不知道自己是否发过请求包。他也不管是否合法的应答，只要收到目标mac地址是自己的ARP reply或者ARP广播包（包括ARP reply和ARP request），都会接受并缓存。本次实验便是尝试ARP攻击进行安全测试

    ARP攻击步骤：

    1. ARP缓存投毒（ARP Cache Poisoning）：攻击者发送伪造的ARP响应包（ARP Reply）来欺骗其他主机，将攻击者的MAC地址与目标主机的IP地址绑定到一起。这样，目标主机的ARP缓存中将保存了错误的映射关系。
    2. 中间人攻击（Man-in-the-Middle Attack）：攻击者成为通信双方之间的中间人，并拦截、篡改或监听通信流量。如此即可实现DoS攻击（Denial-of-Service Attack）和会话劫持（Session Hijacking，窃取受害者的会话信息，如登录凭据、Cookie等，获取非法访问权限）。

    ARP攻击有效条件

    ARP攻击需要满足以下条件：

    1. 与目标网络共享局域网（LAN）：ARP攻击通常发生在局域网环境中，因为ARP协议是一种局域网协议。攻击者需要位于目标网络的同一物理或逻辑子网上，以便发送欺骗性的ARP响应包。
    2. 攻击者得到被攻击者的IP地址：攻击者需要在局域网内不断广播被攻击者的IP地址的MAC地址为自己的MAC地址
    3. 攻击者能够发送伪造的ARP响应包：攻击者需要具备发送和接收ARP数据包的能力，并能够伪造ARP响应包，误导其他主机的ARP缓存。
    4. 目标主机没有对ARP缓存进行正确的验证或保护：如果目标主机没有实施有效的ARP缓存验证机制，就容易受到ARP攻击。这使得攻击者能够发送伪造的ARP响应包，并成功将错误的MAC地址与目标IP地址关联起来。
    5. 目标主机对欺骗的ARP响应包进行接受和处理：目标主机需要接受并处理攻击者发送的伪造的ARP响应包。如果目标主机接受并保存了错误的ARP映射关系，就会在后续的通信中受到影响。
    6. 攻击者位于目标主机和通信目标之间：为了进行中间人攻击或拦截通信流量，攻击者需要位于目标主机和通信目标之间，以便截取、篡改或监听通信。

    ARP攻击危害

    ARP攻击的危害如下：

    1. 中断网络连接：通过ARP攻击，攻击者可以欺骗目标主机将其默认网关或其他重要网络设备的MAC地址与错误的IP地址绑定。这会导致目标主机无法正确路由网络流量，从而中断其与网络的连接。
    2. 窃取敏感信息：ARP攻击者可以利用中间人攻击技术截获通信流量，并窃取敏感信息，例如登录凭据、会话令牌、个人数据等。这可能导致身份盗窃、数据泄露和其他安全问题。
    3. 篡改通信内容：攻击者可以修改经过其控制的通信流量，从而篡改数据包的内容。这可能导致信息被篡改、恶意软件注入或传输错误的指令，破坏通信的完整性和可靠性。
    4. 拒绝服务攻击：通过ARP攻击，攻击者可以欺骗目标主机将其自身的IP地址与另一个合法主机的MAC地址绑定。这可能导致目标主机无法正常连接到网络或与其他设备通信，从而实施拒绝服务（DoS）攻击。
    5. 会话劫持：ARP攻击者可以窃取受害者的会话信息，例如Web应用程序的会话ID或Cookie。这使得攻击者能够伪装成受害者，获取非法访问权限或执行未经授权的操作。

    DNS挟持

    利用ettercap进行中间人攻击之DNS劫持

    实验原理

    DNS劫持：攻击者通过篡改本地DNS缓存或在中间节点欺骗网络中其他设备的DNS请求，将用户的合法DNS查询请求重定向到攻击者控制的恶意网站或服务器上。这样攻击者就可以获取用户的敏感信息，包括账户名、密码等等。例如，当用户在浏览器中输入一个网站的URL时，浏览器会向本地DNS服务器发出DNS查询请求，查询网站的IP地址。如果攻击者劫持了这个请求，就会返回一个虚假的IP地址，将用户重定向到一个假冒网站，从而达到劫持的目的。

    第一部分，使用ettercap图形化界面进行ARP欺骗

    第二部分，使用ettercap进行DNS劫持，开启Apache服务，让目标主机访问任何网站都劫持到kali系统主页

    ARP防御

    ARP防御方法

    1. 使用静态 ARP 表： 在网络中配置静态 ARP 表，将特定 IP 地址与相应的 MAC 地址进行绑定。这样可以防止攻击者通过伪造 ARP 请求来欺骗网络中的主机。
    1
    arp -s 172.20.10.1 0a:c7:29:2d:6d:64

    1. 启用 ARP 监控/防御工具： 使用 ARP 监控和防御工具可以检测和阻止 ARP 攻击。这些工具可以监视网络流量中的异常 ARP 请求，并采取适当的措施进行防御。

    2. 使用网络交换机的安全功能： 使用现代网络交换机， 对 ARP 攻击进行防御，如静态 ARP 表绑定、动态 ARP 检测和防御、ARP Snooping 等。

    3. 启用网络流量加密： 使用加密协议（如 SSL/TLS）来保护网络流量，可以减少 ARP 攻击的风险。加密可以防止攻击者在网络上窃听或篡改 ARP 请求和响应。

    4. 实施网络访问控制： 在网络设备上配置适当的访问控制列表（ACL）或防火墙规则，限制 ARP 请求和响应的流动。仅允许授权的主机进行 ARP 通信，可以减少 ARP 攻击的成功率。

    5. 定期检查和监视网络： 定期检查网络设备、主机和流量，以便及时发现和应对任何潜在的 ARP 攻击。监视网络日志和异常活动，以便及早发现和响应可能的攻击。

    Https拓展

    HTTPS是一种用于在网络上安全传输数据的协议。出于以下几点原因，采用HTTPS协议的网页可以有效防止ARP攻击:

    • 加密通信： HTTPS使用SSL/TLS协议对数据进行加密。即使攻击者能够截获数据包，由于数据已经被加密，他们无法直接读取敏感信息。
    • 身份验证： SSL/TLS协议还提供了身份验证机制，确保通信的两端是合法的，并且通信的数据没有被中间人篡改。这使得攻击者很难在通信过程中插入恶意数据或欺骗受害者。
    • 数字证书： HTTPS使用数字证书来验证服务器的身份。这些证书由信任的证书颁发机构（CA）签发，用于确认服务器是合法的。这防止了攻击者通过ARP欺骗来模拟服务器。

    但是通过Kali Liunx上的sslstrip工具，我们可以将采用https协议传输的网页降级为http协议后，再转发给靶机，从而实现arp监听。具体攻击流程如下所示：

    实现组合攻击

    将ARP欺骗与窃听、DNS挟持和钓鱼网站攻击相结合，利用ARP欺骗作为受害者与网关之间的中间人，窃听流量信息，同时开启DNS挟持，将对方想要访问的正确网站替换为钓鱼网站，其中钓鱼网站和正确的网站十分相似，受害者很容易受到欺骗并输入个人信息。
